# パーソナルダイジェスト(React Native / Expo版)

PWA版と同等機能のネイティブアプリです。**Expo Go**(App Storeの無料アプリ)でiPhone実機動作します。Mac不要・Windowsのみで開発/実行できます。

- Expo SDK **57**(managed workflow, TypeScript)/ React Native 0.86
- ネイティブモジュールの追加ビルド(dev build)不要 — Expo Go互換の構成のみ使用

## 機能

- 取得国の複数選択(米・日・英・中・韓・独・仏・印など18か国)+ ニュース表示言語の独立選択
- Google News RSSからのフィード取得(見出し+要約カード → タップで詳細シート → 元記事リンク)
- 好きな時刻・カテゴリの**ローカル通知**(expo-notifications、毎日繰り返し)
- 閲覧/スキップの**端末内学習**による推薦(AsyncStorage保存、時間減衰つき重み付け)
- オフライン閲覧: 既読記事+「ワンステップ成長」上位10件を自動キャッシュ、フィードも15分キャッシュ
- オンボーディング、設定画面、ダークモード(自動/手動)

## WindowsでのセットアップとiPhone(Expo Go)での実機確認手順

### 0. 前提
- Windows PC と iPhone が**同じWi-Fi**に接続されていること
- iPhoneに App Store から **Expo Go** をインストール

### 1. Node.js のインストール(未導入の場合)
https://nodejs.org/ja から **LTS版** をダウンロードしてインストール。
確認: コマンドプロンプト(またはPowerShell)で
```
node -v
```
がバージョンを表示すればOK(v20以上推奨)。

### 2. プロジェクトの取得
Git がある場合:
```
git clone https://github.com/SHOTARO-Tukeyleg/news-digest-native.git
cd news-digest-native
```
Git がない場合はGitHubリポジトリの「Code → Download ZIP」でダウンロードして展開し、そのフォルダでコマンドプロンプトを開く。

### 3. 依存パッケージのインストール
```
npm install
```

### 4. 開発サーバー起動
```
npx expo start
```
ターミナルに**QRコード**が表示されます。

### 5. iPhoneで開く
iPhoneの**カメラアプリ**でQRコードを読み取る → 「Expo Goで開く」をタップ。
初回はJSバンドルの転送に数十秒かかります。

うまく接続できない場合(会社Wi-Fi等でPCとiPhoneが隔離されている場合):
```
npx expo start --tunnel
```
でトンネル接続を試してください(ngrok経由、少し遅くなります)。

## Expo Goでの制約(正直な説明)

- **ローカル通知は動作します**(iOS/Android)。毎日指定時刻に繰り返し通知が届きます。
- **リモートプッシュ通知は不可**: Expo GoではSDK 53以降リモートプッシュがサポートされません(自前ビルド=development build / EAS Buildが必要)。本アプリはローカル通知のみ使用しているため影響ありません。
- **通知の本文は定型文**です。iOSは通知発火時にアプリのJSを実行できないため、「朝のダイジェストが届いています」という通知が指定時刻に届き、**具体的な記事はアプリを開いた瞬間に生成**されてバナー表示されます(1日1回)。
- 通知許可はiOSの場合、初回に表示されるダイアログで「許可」を選んでください。あとから変更する場合は iPhoneの設定 → Expo Go → 通知。
- Expo Goを完全終了(スワイプで削除)しても、スケジュール済みローカル通知はOSに登録されているため届きます。ただしExpo Go内で**別のプロジェクトを開くと通知登録がリセットされる**ことがあります。

## ニュース取得について

ネイティブの`fetch`には**ブラウザのCORS制約が存在しない**ため、Google News RSSを**直接取得**します(PWA版で必要だったrss2jsonプロキシは、直接取得が失敗した場合のフォールバックとしてのみ残しています)。プロキシ依存が大幅に減り、取得の信頼性はPWA版より高い構成です。

RSSのdescriptionは記事本文ではなく見出し+関連報道リンクのため、「全文」は元記事リンク(Safari等で開く)で提供します。オフライン時に読めるのは見出し・要約・関連リンクです。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `App.tsx` | UI全体(タブ、ホーム/成長/保存/設定、詳細モーダル、オンボーディング) |
| `core.ts` | 純粋ロジック(RSSパーサ、URL生成、トークナイザ、学習・スコアリング)— Nodeでテスト可能 |
| `data.ts` | 国・言語・カテゴリの静的データ |
| `feed.ts` | フィード取得(直接fetch→rss2jsonフォールバック、15分キャッシュ) |
| `storage.ts` | AsyncStorageヘルパー |
| `notify.ts` | ローカル通知(毎日繰り返し+即時) |
| `i18n.ts` / `theme.ts` | UI文言(日/英)・カラーテーマ |
